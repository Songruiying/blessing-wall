# 祝福墙签到抽奖系统

一个完整的签到、祝福墙展示和抽奖系统，支持学生组和老师组分别抽奖。

## 功能特点

1. **扫码签到页面** (`index.html`)
   - 输入姓名和身份（学生组/老师组）
   - 输入祝福语（最多100字）
   - 实时字符计数
   - 提交成功提示

2. **祝福墙展示页面** (`wall.html`)
   - 大屏幕展示所有祝福
   - 信封飞入动画效果
   - 学生组和老师组不同颜色区分
   - 实时统计显示
   - 自动检测新祝福并添加

3. **抽奖页面** (`lottery.html`)
   - 支持学生组和老师组分别抽奖
   - 抽奖动画效果
   - 结果展示（姓名、祝福语、组别）
   - 语音播报（浏览器支持时）
   - 彩纸庆祝效果

## 使用方法

1. **签到页面**：参与者扫码打开 `index.html`，填写信息并提交祝福
2. **祝福墙**：在大屏幕上打开 `wall.html`，实时显示所有祝福
3. **抽奖页面**：在抽奖环节打开 `lottery.html`，选择组别进行抽奖

## 技术说明

- 使用 `localStorage` 存储祝福数据（实际项目中应使用后端API）
- 纯前端实现，无需服务器
- 响应式设计，支持各种屏幕尺寸
- 跨标签页数据同步（通过 `storage` 事件）

## 数据格式

祝福数据存储在 `localStorage` 中，格式如下：

```json
[
  {
    "id": 1234567890.123,
    "name": "张三",
    "group": "student",
    "blessing": "祝学院越来越好！",
    "timestamp": "2024-01-01T12:00:00.000Z"
  }
]
```

## 部署建议

1. 将文件部署到Web服务器
2. 将 `localStorage` 替换为后端API
3. 使用WebSocket实现实时数据同步
4. 添加用户认证和权限控制

## 浏览器兼容性

- Chrome/Edge（推荐）
- Firefox
- Safari
- 移动端浏览器

## 注意事项

- 当前使用 `localStorage` 存储，数据仅保存在本地浏览器
- 如需多设备同步，请使用后端API
- 语音播报功能需要浏览器支持 Web Speech API
